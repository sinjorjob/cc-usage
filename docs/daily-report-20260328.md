# Daily Report - 2026-03-28

## セッション概要: コンテキスト空間可視化ダッシュボードの設計

### 要件の発端
- `/context`コマンドの出力がテキストベースで見づらい
- cc-usageガジェットにコンテキスト可視化機能を追加したいという要望

---

## 意思決定の流れ

### 1. 初期設計 → cc-usageガジェットへの組み込み案
- 200x200のCompactモードからダブルクリックで420x520のDashboardモードに展開する案
- ドーナツチャート + カテゴリ凡例 + 詳細タブ（MCP/Memory/Skills）

### 2. データ取得方法の検討
- **Hook機能が企業PCで使えない**制約が判明
- ファイル監視、クリップボード監視、手動ペースト等を検討
- → `claude -p "/context"` でCLIから直接取得可能と判明、これをメイン方式に決定

### 3. 独立アプリ化の決断
- ガジェットへの組み込みは「サイズが大きくなりすぎる」「操作ミスでダッシュボードモードになるとウザい」
- → **cc-usageとは別の独立Electronアプリ** (`cc-context-viewer/`) として開発する方針に

### 4. データソース設計の確定
- プロバイダー抽象化パターン: MockProvider（開発用）→ CliProvider（本番用）に差し替え可能
- **定期自動更新は不要**: 手動更新ボタン + 起動時取得のみ

---

## UIデザインの進化

### 3ビュー構成に決定

| ビュー | 説明 | 用途 |
|--------|------|------|
| **Chart** | ドーナツチャート + 凡例 + サマリーカード + 詳細タブ | 精密な数値確認 |
| **Space Map** | Treemap（面積＝トークン量） | 占有比率を面積で一発把握 |
| **Cosmos** | 星雲ベースの宇宙空間可視化（デフォルト表示） | 世界観としての空間認識 |

### Cosmosビューの特徴
- 各カテゴリ＝星雲、サイズ＝トークン消費量（power 0.5スケーリング）
- 有機的な脈動（複数sin波重ね）、Lissajous軌道での漂流
- ガス雲の回転、フィラメントの揺れ、コアの明滅
- Orbitronフォント（英語） + Noto Sans JP（日本語）
- 英語ラベルがデフォルト、ホバーで日本語説明（枠なしフェードイン）
- 外周リングゲージで全体使用率を表示
- たまに流れ星が横切る演出

### 惑星化の試み → リバート
- 落合陽一的な惑星 + 軌道運動 + デジタルグリッドスタイルを試みた
- → 見た目がチープになったためリバートし、星雲バージョンに戻した

---

## 確定した設計方針

- **独立Electronアプリ**として開発（cc-usageとは別プロジェクト）
- **データ取得**: `claude -p "/context"` + パーサー + プロバイダー抽象化
- **3ビュー切替**: Chart / Space Map / Cosmos（Cosmosがデフォルト）
- **Hook不使用**、手動更新のみ
- **日本語対応**: Cosmos内テキスト（コンテキスト空間、使用率等）は日本語
- **フォント**: Noto Sans JP（日本語・ラベル） + Orbitron（英語装飾）
